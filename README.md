# 360° Feedback Bot

Система автоматизированного сбора и анализа обратной связи по методу 360°.  
Telegram-бот проводит структурированный опрос, ИИ анализирует ответы и формирует персональный HTML/PDF-отчёт.

Разработана Корпоративным Университетом ГК «КОРУС Консалтинг» совместно с **KORUS Brain Up**.

---

## Раздел 1. Функциональность

### Что такое оценка 360°?

Оценка 360° — это метод получения обратной связи о сотруднике от нескольких групп людей одновременно: его руководителя, коллег, подчинённых и от него самого (самооценка). Это позволяет получить объективную, многостороннюю картину сильных сторон и зон развития сотрудника.

### Как работает система

Процесс состоит из нескольких шагов:

1. **HR-специалист создаёт кампанию** в веб-панели администратора: задаёт название, список компетенций и добавляет участников (респондентов) с указанием их ролей.
2. **Система автоматически формирует персональные ссылки** для каждого участника.
3. **Респондент переходит по ссылке** — она открывает Telegram-бот, который проводит опрос.
4. **Бот общается с респондентом** в удобном чат-формате: задаёт вопросы, принимает оценки и развёрнутые комментарии.
5. **После завершения всех опросов** ИИ анализирует собранные данные и формирует отчёт.
6. **HR и оцениваемый сотрудник** получают готовый структурированный отчёт с выводами и рекомендациями.

---

### Роли участников

В каждой кампании можно добавить участников пяти ролей:

| Роль | Описание |
|------|----------|
| **Самооценка** | Сам оцениваемый сотрудник. Проходит только оценку компетенций, без открытых вопросов. |
| **Руководитель** | Непосредственный или функциональный руководитель. |
| **Коллеги** | Сотрудники одного уровня, работающие рядом. |
| **Клиенты** | Внутренние или внешние клиенты. |
| **Сотрудники** | Подчинённые оцениваемого. |

Для роли «Самооценка» предусмотрен отдельный сценарий опроса: бот задаёт те же вопросы по компетенциям, но переформулирует их от первого лица («насколько это поведение проявляется в твоей работе?») и не задаёт открытые вопросы.

Ответы коллег, клиентов и сотрудников остаются **анонимными** — в отчёте они объединяются без привязки к конкретному человеку.

### Опрос через Telegram

Каждый участник получает персональную ссылку вида `t.me/ВашБот?start=токен`. При переходе по ней открывается Telegram, и бот сразу начинает опрос.

#### Сценарий для коллег, руководителя, клиентов, сотрудников:

```
Приветствие с объяснением цели опроса и кнопкой [Начать]
  └─ Оценка компетенций (кнопки 1 / 2 / 3 / 4 / 5 / N/A)
      └─ Открытые вопросы (текстом или голосовым сообщением)
          └─ Завершение — автоматический запуск анализа
```

#### Сценарий для роли «Самооценка»:

```
Приветствие с просьбой оценить себя и кнопкой [Начать]
  └─ Оценка компетенций (кнопки 1 / 2 / 3 / 4 / 5 / N/A)
      └─ Завершение — опрос сразу закрыт, без открытых вопросов
```

#### Оценка компетенций

По каждой компетенции бот показывает её название, блок и описание. Респондент нажимает одну кнопку:

```
1 — Почти никогда
2 — Редко
3 — Иногда
4 — Часто
5 — Почти всегда
N/A — Не было возможности наблюдать
```

#### Открытые вопросы

После оценки всех компетенций бот задаёт четыре открытых вопроса:

1. **Сильные стороны** — что выделяется в позитивном смысле?
2. **Зоны развития** — что стоит начать делать или делать чаще?
3. **Поведение, которое мешает** — что стоит перестать делать?
4. **Дополнительно** (необязательно) — свободный комментарий; при желании можно подписаться.

Отвечать можно **текстом или голосовым сообщением** — бот автоматически распознаёт голос и переводит его в текст.

#### Дополнительные команды бота

| Команда | Действие |
|---------|----------|
| `/start <токен>` | Начать опрос по персональной ссылке |
| `/resume` | Продолжить с последнего шага |
| `/finish` | Завершить опрос досрочно |
| `/help` | Справка по командам |

### Веб-панель администратора

HR-специалист управляет всем через удобный веб-интерфейс:

#### Список кампаний
- Создание новой кампании и просмотр всех существующих.
- Быстрый статус каждой кампании (черновик / активна / завершена).

#### Настройки кампании
- Название, описание, ФИО оцениваемого сотрудника.
- Статус кампании (черновик → активна → завершена → архив).
- **Приветственное и завершающее сообщения для каждой роли** — можно написать разный текст для руководителя, коллег, самооценки и т.д. В тексте поддерживается плейсхолдер `<оцениваемый сотрудник>`, который автоматически заменяется на ФИО.
- **Импорт текстов из другой кампании** — одним кликом скопировать тексты сообщений из ранее настроенной кампании.

#### Компетенции
- Список компетенций с возможностью редактировать, включать/отключать и менять порядок.

#### Респонденты
- Добавление участников с указанием роли, должности, отдела.
- Генерация персональной invite-ссылки для каждого участника.
- Статус каждого участника: приглашён / начал / завершил.

#### Прогресс
- Сводная таблица по всем участникам: кто прошёл, кто ещё нет.

#### Отчёт
- Просмотр визуального HTML-отчёта прямо в браузере.
- Скачивание отчёта в формате **HTML** (для архива).
- Печать / сохранение в **PDF** через браузер.
- Кнопка «Перегенерировать отчёт» — повторный запуск анализа ИИ.

> Уведомление оцениваемому в Telegram отправляется **только после** успешного формирования AI-анализа.

### Отчёт 360°

Отчёт формируется автоматически после завершения всех опросов (или по кнопке) и содержит:

#### Обложка
- ФИО оцениваемого сотрудника, название кампании, дата формирования.
- Логотипы организации (КОРУС Консалтинг и KORUS Brain Up).

#### Сводная статистика
- Таблица завершивших респондентов по ролям.

#### Паутинка (Radar Chart)
- Визуальная диаграмма средних оценок по каждой компетенции.
- Отдельные линии для каждой роли с цветовым кодированием.

#### Таблица средних оценок
- Числовые средние по каждой компетенции в разрезе ролей.
- Выделение наивысших и наименьших оценок.

#### Сильные стороны / Потребности в развитии (квадрант)
- Scatter-диаграмма: ось X — самооценка, ось Y — оценка другими.
- Пересечение осей в точке 3,75/3,75 делит компетенции на 4 зоны:
  - **Очевидные сильные стороны** — высокая самооценка + высокая оценка другими.
  - **Скрытые сильные стороны** — высокая оценка другими, но низкая самооценка.
  - **Зоны развития** — низкая оценка и самооценка.
  - **Слепые пятна** — высокая самооценка, но низкая оценка другими.
- Под квадрантом — таблица с компетенциями в каждой зоне и её описанием.

#### Детальные оценки по компетенциям
- Для каждой компетенции: числовые оценки по всем ролям.

#### Обратная связь
- Ответы на открытые вопросы от всех респондентов, сгруппированные по вопросам.

#### Анализ оценки 360° (AI)
- Автоматически формируется при переводе кампании в статус «Завершена».
- GPT-4o анализирует средние оценки по всем ролям и ответы на открытые вопросы.
- Структура анализа: **На что обратить внимание** → **Что требует уточнений и прояснений** → **Компетенции к развитию**.
- Раздел завершается дисклеймером: *Анализ и рекомендации сформированы искусственным интеллектом.*

#### Публичная ссылка на отчёт
- При переводе кампании в статус «Завершена» автоматически создаётся публичная ссылка на отчёт **без необходимости входа в систему**.
- Ссылка отправляется оцениваемому сотруднику в Telegram — **после** того, как AI-анализ успешно сформирован.

## Раздел 2. Архитектура и реализация

### Общая схема

```
Telegram Bot API
      │  webhook (POST /api/telegram/webhook)
      ▼
  Vercel (Next.js 15 — serverless)
  ┌────────────────────────────────────────────────┐
  │  /api/telegram/webhook   приём сообщений        │
  │  /api/cron/jobs          фоновые задачи (cron)  │
  │  /api/campaigns/[id]     управление кампаниями   │
  │  /api/reports/[token]    публичный отчёт         │
  │  /(admin)/campaigns      веб-панель HR           │
  └────────────────────────────────────────────────┘
        │                          │
        ▼                          ▼
  PostgreSQL (Supabase)       OpenAI API
  Prisma ORM                  GPT-4o + Whisper
```

### Стек технологий

| Слой | Технология | Роль |
|------|-----------|------|
| **Фронтенд / Бэкенд** | Next.js 15 App Router + TypeScript | Единое приложение: веб-панель + API |
| **База данных** | PostgreSQL + Prisma ORM | Хранение всех данных, схема, миграции |
| **Telegram** | Telegram Bot API (webhook) | Приём сообщений, отправка ответов, inline-кнопки |
| **ИИ — интервью** | OpenAI GPT-4o | Принятие решений в диалоге с респондентом |
| **ИИ — анализ** | OpenAI GPT-4o (JSON mode) | Извлечение структурированной обратной связи по компетенциям |
| **ИИ — интерпретация** | OpenAI GPT-4o (text mode) | Анализ 360°: на что обратить внимание, рекомендации к развитию |
| **ИИ — голос** | OpenAI Whisper | Транскрипция голосовых сообщений |
| **Аутентификация** | bcryptjs + jose (JWT в cookie) | Вход в панель администратора |
| **Валидация** | Zod | Проверка входных данных на всех границах |
| **Графики** | Chart.js 4 (inline CDN) | Паутинка и квадрант в HTML-отчёте |
| **Тесты** | Vitest | Юнит-тесты бизнес-логики |

### База данных

Все данные хранятся в PostgreSQL (деплой — Supabase). Схема управляется через Prisma.

| Таблица | Содержимое |
|---------|-----------|
| `admins` | Учётные записи администраторов (HR) |
| `campaigns` | Кампании 360°: название, статус, настройки, публичный токен отчёта, AI-анализ |
| `campaign_role_messages` | Кастомные тексты приветствия/завершения для каждой роли |
| `competency_templates` | Глобальная библиотека шаблонов компетенций |
| `competencies` | Компетенции конкретной кампании |
| `respondents` | Участники: роль, статус, invite-токен, Telegram ID |
| `interview_sessions` | Сессия опроса с текущим состоянием (JSON state machine) |
| `messages` | Лог переписки: текст, голос, системные сообщения |
| `competency_ratings` | Числовые оценки 1–5/N/A по каждой компетенции |
| `competency_feedback` | Извлечённая ИИ обратная связь по компетенции |
| `analysis_reports` | Финальные отчёты (Markdown + JSON) |
| `jobs` | Очередь фоновых задач с retry |
| `telegram_update_logs` | Журнал обработанных обновлений (идемпотентность webhook) |

Все записи, связанные с кампанией, удаляются каскадно при удалении кампании.

### Telegram: обработка сообщений

Telegram отправляет каждое новое сообщение на webhook-URL (`/api/telegram/webhook`) через HTTPS POST. Система:

1. Проверяет подпись (`X-Telegram-Bot-Api-Secret-Token`).
2. Определяет тип обновления: команда (`/start`), callback от кнопки, текст, голос.
3. Загружает контекст респондента из БД по `telegramUserId`.
4. Выполняет логику шага опроса (state machine).
5. Отправляет ответ обратно в Telegram через REST API.

Голосовые сообщения: бот скачивает аудиофайл, транскрибирует через Whisper API, сохраняет текст.

### Фоновые задачи

| Задача | Когда ставится | Что делает |
|--------|---------------|-----------|
| `extract_feedback` | После завершения каждого опроса | ИИ анализирует переписку, извлекает структурированную обратную связь по каждой компетенции (JSON) |
| `generate_reports` | При ручном запуске через панель | ИИ формирует финальный отчёт по всей кампании |
| `generate_ai_analysis` | При переводе кампании в статус «Завершена» | GPT-4o формирует текстовый анализ 360° с рекомендациями; по завершении отправляет Telegram-уведомление оцениваемому |

Очередь обрабатывается по cron-расписанию (ежедневно в 09:00 МСК) и сразу после каждого опроса. Задачи имеют механизм retry при ошибках и soft-lock для защиты от дублирования.

### HTML-отчёт

Отчёт формируется как **самодостаточный HTML-файл**: все стили и изображения встроены inline (base64). Это означает, что файл можно открыть на любом устройстве без интернета и он будет выглядеть одинаково.

Графики (паутинка, квадрант) отрисовываются через Chart.js, подгружаемый с CDN при первом открытии.

### Деплой

| Компонент | Платформа |
|-----------|-----------|
| **Приложение** | [Vercel](https://vercel.com) — serverless, автодеплой из GitHub при push в `main` |
| **База данных** | [Supabase](https://supabase.com) — управляемый PostgreSQL |
| **Cron** | Vercel Cron Jobs (настроены в `vercel.json`, запуск ежедневно в 09:00) |
| **Webhook** | Telegram Bot API → Vercel serverless function |

### Безопасность

- Вход в панель администратора защищён паролем (bcryptjs) + сессионной cookie (JWT, 7 дней).
- Telegram webhook проверяет секретный заголовок (`X-Telegram-Bot-Api-Secret-Token`).
- Rate limiting: не более 50 запросов за 60 секунд с одного Telegram-пользователя.
- Пользовательский ввод санитизируется перед подстановкой в промпты ИИ.
- Публичные отчёты доступны только по случайному UUID-токену.
- Zod-валидация на всех точках входа: формы, API, webhook.

## Раздел 3. Инструкция по развёртыванию

### Что понадобится

- Аккаунт [GitHub](https://github.com)
- Аккаунт [Vercel](https://vercel.com) (бесплатный план подходит)
- Аккаунт [Supabase](https://supabase.com) (бесплатный план: 500 МБ PostgreSQL)
- Telegram-бот (создаётся через [@BotFather](https://t.me/BotFather))
- API-ключ [OpenAI](https://platform.openai.com) (нужны доступ к GPT-4o и Whisper)

---

### Шаг 1. База данных (Supabase)

1. Зарегистрируйтесь на [supabase.com](https://supabase.com) и создайте новый проект.
2. В разделе **Project Settings → Database** найдите строки подключения:
   - **Connection string (Transaction mode)** → это `DATABASE_URL`
   - **Connection string (Session mode / Direct)** → это `DIRECT_URL`
3. Сохраните обе строки — они понадобятся при настройке Vercel.

---

### Шаг 2. Telegram-бот

1. Напишите [@BotFather](https://t.me/BotFather) и создайте бота командой `/newbot`.
2. Сохраните **токен бота** (вида `123456:ABC-DEF...`).
3. Придумайте случайную строку для `TELEGRAM_WEBHOOK_SECRET` (например, UUID).
4. Имя бота (`@username`) сохраните для переменной `TELEGRAM_BOT_USERNAME`.

---

### Шаг 3. Форк и деплой на Vercel

1. Сделайте форк этого репозитория на GitHub.
2. Зайдите в [vercel.com](https://vercel.com), нажмите **Add New Project** и выберите свой форк.
3. В разделе **Environment Variables** добавьте все переменные из таблицы ниже.
4. Нажмите **Deploy** — Vercel соберёт и опубликует приложение.
5. Скопируйте URL вашего приложения (например, `https://my-360-bot.vercel.app`).

#### Переменные окружения

| Переменная | Обязательная | Описание |
|-----------|:---:|----------|
| `DATABASE_URL` | ✅ | PostgreSQL connection string — Transaction mode (Supabase) |
| `DIRECT_URL` | ✅ | PostgreSQL direct connection string — Session mode (Supabase) |
| `OPENAI_API_KEY` | ✅ | API-ключ OpenAI |
| `TELEGRAM_BOT_TOKEN` | ✅ | Токен вашего Telegram-бота |
| `TELEGRAM_WEBHOOK_SECRET` | ✅ | Случайная строка для проверки подписи webhook |
| `TELEGRAM_BOT_USERNAME` | — | Username бота без `@` (для красивых invite-ссылок) |
| `APP_BASE_URL` | ✅ | Полный URL приложения (`https://...vercel.app`) |
| `AUTH_SECRET` | ✅ | Случайная строка для подписи сессионной cookie (мин. 32 символа) |
| `CRON_SECRET` | ✅ | Секрет для защиты cron-эндпоинта |
| `KV_REST_API_URL` | — | Upstash Redis URL (опционально, для rate limiting) |
| `KV_REST_API_TOKEN` | — | Upstash Redis Token (опционально) |

> Без `KV_REST_API_URL` система работает с fallback на PostgreSQL — это нормально для старта.

### Шаг 4. Применение миграций БД

После первого деплоя нужно создать таблицы в базе данных. Выполните локально:

```bash
# Клонируйте ваш форк
git clone https://github.com/ВАШ_АККАУНТ/360-Feedback-bot.git
cd 360-Feedback-bot

# Установите зависимости
npm install

# Скопируйте .env.example в .env и заполните переменные (те же, что в Vercel)
cp .env.example .env

# Примените миграции к вашей Supabase БД
npx prisma migrate deploy

# Создайте начальные данные (аккаунт администратора и шаблоны компетенций)
npm run prisma:seed
```

После seed в системе будет создан аккаунт администратора:
- **Email:** `admin@360bot.local`
- **Пароль:** `ChangeMe123!`

> ⚠️ Сразу смените пароль после первого входа!

### Шаг 5. Настройка Telegram webhook

Выполните команду, подставив ваши значения:

```bash
curl -X POST "https://api.telegram.org/bot<ТОКЕН_БОТА>/setWebhook" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://ВАШ_ДОМЕН.vercel.app/api/telegram/webhook",
    "secret_token": "<TELEGRAM_WEBHOOK_SECRET>"
  }'
```

Проверьте, что webhook установлен корректно:

```bash
curl "https://api.telegram.org/bot<ТОКЕН_БОТА>/getWebhookInfo"
```

В ответе должен быть ваш URL и `"pending_update_count": 0`.

### Шаг 6. Первый вход и создание кампании

1. Откройте `https://ВАШ_ДОМЕН.vercel.app/login`.
2. Войдите с данными администратора.
3. Нажмите **Создать кампанию**, заполните ФИО оцениваемого сотрудника и другие поля.
4. В разделе **Респонденты** добавьте участников и скопируйте их invite-ссылки.
5. Отправьте ссылки участникам — они автоматически откроют Telegram-бота.
6. Переведите кампанию в статус **active**, чтобы бот принимал ответы.
7. После завершения всех опросов нажмите **Перегенерировать отчёт** в разделе Отчёт.

### Локальная разработка

```bash
npm install
cp .env.example .env
# заполнить .env

npx prisma migrate dev   # создать/обновить схему локально
npm run prisma:seed      # тестовые данные

npm run dev              # запуск на http://localhost:3000
```

> **Windows:** при сборке автоматически применяется патч `scripts/readlink-patch.cjs`, исправляющий поведение `fs.readlink` в webpack. На Linux и в Vercel патч не нужен и не применяется.

Запуск тестов:

```bash
npm test
```

Линтинг и проверка сборки:

```bash
npm run lint
npm run build
```

### Обновление системы

При изменениях в схеме БД после обновления кода выполните:

```bash
npx prisma migrate deploy
```

Vercel автоматически пересобирает и деплоит приложение при каждом push в `main`. Миграции нужно применять вручную (или через CI/CD скрипт).
